[추가 결정] 긴 텍스트 처리 — A 옵션 sub-option

A안(picker 별도 줄) 채택 후, "원본 → 마스킹" 텍스트 자체가 모달 폭을 넘는 경우 4가지 처리 방식. 예: 매우 긴 이메일 / 매우 긴 주소.

A.W1. wrap (break-words) — 자동 줄바꿈 ⭐ 추천

  • 사람 이름 조성도→홍길동
  • 이메일 verylongemailaddress.with.dots@super-long-domain-name.example.com→v***************************************************@super-long-domain-name.example.com
  • 주소 서울특별시 영등포구 양평로 28길 5번지, 미래빌딩 103동 802호→●●●●●●●●●●●●●●●●●●●●●●●●●●●●●●●

✅ 정보 잘림 X. 행 키 가변 (텍스트 길이에 따라 자연스럽게 늘어남). 짧은 텍스트는 한 줄, 긴 텍스트는 2-3줄. 가장 단순한 구현 (truncate → break-words 한 글자 변경).

A.W2. 원본 / 마스킹 별도 줄 분리

  • 사람 이름 조성도
    → 홍길동
  • 이메일 verylongemailaddress.with.dots@super-long-domain-name.example.com
    → v***************************************************@super-long-domain-name.example.com
  • 주소 서울특별시 영등포구 양평로 28길 5번지, 미래빌딩 103동 802호
    → ●●●●●●●●●●●●●●●●●●●●●●●●●●●●●●●

원본은 한 줄 truncate(공간 절약), 마스킹은 wrap(전체 보기). 시각적 구분 명확. 행 키 +20px 추가.

A.W3. truncate 유지 + 호버 tooltip

  • 사람 이름 조성도→홍길동
  • 이메일 verylongemailaddress.with.dots@super-long-domain-name.example.com→v***************************************************@super-long-domain-name.example.com
  • 주소 서울특별시 영등포구 양평로 28길 5번지, 미래빌딩 103동 802호→●●●●●●●●●●●●●●●●●●●●●●●●●●●●●●●

한 줄 유지(컴팩트). 마우스 hover 시 full text tooltip. ⚠️ 모바일/터치 환경에서 hover 동작 X. textarea 결과로도 확인 가능하니 보조 수단.

A.W4. 가운데 ellipsis (start ··· end)

  • 사람 이름 조성도→홍길동
  • 이메일 verylongemai…example.com→v***********…example.com
  • 주소 서울특별시 영등포구…미래빌딩 103동 802호→●●●●●●●…●●●●●●●●

앞·뒤만 살리고 가운데 생략(`···`). 시작·끝이 PII 인식에 유용 (이메일 prefix/domain, 주소 시·도/동호수). 단 JS로 위치 계산 필요 (CSS 단독 불가).
